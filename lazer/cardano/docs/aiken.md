# Aiken — On-chain validator

## What is Aiken?

Aiken is a programming language for Cardano smart contracts. It compiles to
Plutus Core (UPLC). This project uses PlutusV3.

## Project structure

```
on-chain/
  aiken.toml                  ← project config
  validators/
    price_validator.ak        ← the validator
  lib/
    pyth_types.ak             ← OracleDatum type definition
  plutus.json                 ← compiled output (generated by aiken build)
```

## The validator — `price_validator.ak`

### Types

```aiken
// lib/pyth_types.ak
pub type OracleDatum {
  AnyPrice                                       // constructor 0
  MinPrice { min_price_usd_cents: Int }          // constructor 1
  MaxPrice { max_price_usd_cents: Int }          // constructor 2
  PriceRange { lo_cents: Int, hi_cents: Int }    // constructor 3
}
```

### Validation logic

```aiken
validator price_validator(pyth_id: PolicyId) {
  spend(datum, _redeemer, _input, self) {
    expect Some(condition) = datum
    let price_cents = read_ada_usd_cents(pyth_id, self)

    when condition is {
      AnyPrice -> True
      MinPrice { min_price_usd_cents } -> price_cents >= min_price_usd_cents
      MaxPrice { max_price_usd_cents } -> price_cents <= max_price_usd_cents
      PriceRange { lo_cents, hi_cents } ->
        price_cents >= lo_cents && price_cents <= hi_cents
    }
  }
  else(_) { fail }
}
```

The `read_ada_usd_cents` helper:
1. Calls `pyth.get_updates(pyth_id, self)` to read the Pyth Lazer update from
   the transaction's withdrawal redeemer
2. Finds the ADA/USD feed (feed ID 16) in the update
3. Converts mantissa × 10^(exponent+2) to USD cents using rational arithmetic

### Requirements for a valid spend transaction

- Pyth state UTxO as a **reference input**
- Zero-lovelace withdrawal from the Pyth withdraw script with the signed
  update as the **redeemer**
- The price must satisfy the datum condition

### Compile

```bash
cd on-chain
aiken build
```

Generates `plutus.json`. The validator title is
`price_validator.price_validator.spend`.

## Encoding the datum with Mesh SDK

```typescript
function encodeDatum(datum: OracleDatum): object {
  switch (datum.kind) {
    case "AnyPrice":
      return { alternative: 0, fields: [] };
    case "MinPrice":
      return { alternative: 1, fields: [datum.minPriceUsdCents.toString()] };
    case "MaxPrice":
      return { alternative: 2, fields: [datum.maxPriceUsdCents.toString()] };
    case "PriceRange":
      return {
        alternative: 3,
        fields: [datum.loCents.toString(), datum.hiCents.toString()],
      };
  }
}
```

## Lock → Spend flow

```
LOCK
  Wallet → 2 ADA → Script address
  Datum inline: OracleDatum (e.g. MinPrice { min_price_usd_cents: 50 })

  Cardano evaluates: no script, just UTxO creation

SPEND
  Wallet spends the script UTxO
  Redeemer: raw Pyth Lazer payload
  Reference input: Pyth state UTxO
  Withdrawal: Pyth withdraw script (zero-lovelace, signed update)

  Cardano evaluates price_validator.spend:
    ✓ Pyth update verified via pyth library
    ✓ ADA/USD price satisfies datum condition
  → UTxO released, funds to wallet
```
